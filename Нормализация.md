Описание предметной области

Основными участниками бизнес-процесса являются пользователи (далее — покупатели), которые регистрируются в системе, просматривают каталог товаров, выбирают необходимые позиции, добавляют их в заказ и оформляют покупку.

После оформления заказа покупатель производит оплату, после чего осуществляется доставка товара по указанному адресу.

Для корректного функционирования системы необходимо хранить информацию о пользователях, товарах, категориях товаров, заказах, позициях заказов, оплатах и доставке.

0. Сущности предметной области

В рамках предметной области были выделены следующие сущности:

Пользователь

Адрес доставки

Категория товара

Товар

Заказ

Позиция заказа

Оплата

1. Нормализация данных 1НФ

На первоначальном этапе вся информация могла храниться в одной таблице.
При таком подходе возникают следующие проблемы:

данные о товарах повторяются в разных заказах;

возможны неатомарные значения (списки товаров в одном поле);

невозможно добавить пользователя или товар без создания заказа.

Данная структура не соответствует требованиям первой нормальной формы.

2. 2НФ

Для устранения частичных зависимостей данные были разделены на логические сущности:

Заказы

Товары

Позиция заказа

Каждая таблица стала содержать только те данные, которые относятся к соответствующей сущности.

3. 3НФ

На заключительном этапе были устранены транзитивные зависимости.
Итоговая структура базы данных состоит из следующих сущностей:

Пользователи

Адреса доставки

Категории товаров

Заказы

Позиции заказов

Оплаты

В результате каждая таблица содержит атрибуты, напрямую зависящие от первичного ключа, что соответствует требованиям третьей нормальной формы.
